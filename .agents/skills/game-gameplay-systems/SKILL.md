# Game Gameplay Systems

Make the playable loop concrete.

## Specify

- what the player repeatedly does
- how risk and reward work
- progression gates
- win and lose conditions
- restart or retry flow
- difficulty pacing for the first version

## Watch For

- mechanics listed without a real loop
- progression that requires systems not in scope
- a spec that reads like feature soup instead of a playable game
